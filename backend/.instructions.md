# Backend Copilot Overlay

Use this file with root `.instructions.md`.
This file narrows guidance specifically for `backend/` implementation work.

## Scope

- Flask API routes, ML service integration, and surveillance/clustering logic.
- App-factory architecture with config-driven thresholds.

## Implementation defaults

- Keep HTTP parsing/response behavior in `backend/app/api/`.
- Keep reusable domain and ML logic in `backend/app/services/`.
- Read thresholds and tunables from `backend/app/config.py` (or env vars), not hardcoded literals in route conditions.
- Preserve existing API response contracts unless a change is explicitly requested.

## Canonical references

- Entry point: `backend/run.py`
- App factory + blueprint wiring: `backend/app/__init__.py`
- API modules: `backend/app/api/`
- Services: `backend/app/services/`
- Config: `backend/app/config.py`

## Validation

- Run targeted tests nearest the changed behavior in `backend/tests/`.
- Typical smoke check: `python -m pytest backend/tests/test_flask.py -v`
- **CI/CD:** Automatically runs on every PR via `.github/workflows/backend-checks.yml` (pytest, linting, Flask startup check).
