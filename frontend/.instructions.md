# Frontend Copilot Overlay

Use this file with root `.instructions.md`.
This file narrows guidance specifically for `frontend/` implementation work.

## Scope

- Next.js App Router frontend in TypeScript.
- Server actions, Zod schemas, Prisma reads/writes, Supabase auth, and clinician/patient UI flows.

## Implementation defaults

- Prefer Server Components for initial data fetching.
- Use Server Actions for mutations via `next-safe-action`.
- Pair mutation inputs with Zod schemas in `frontend/schemas/`.
- Keep backend URL usage centralized through `frontend/utils/backend-url.ts`.
- Keep mutation outputs structured (`success`/`error`) for resilient UX flows.

## Canonical references

- Action client: `frontend/actions/client.ts`
- Diagnosis flow: `frontend/actions/run-diagnosis.ts`
- Schema examples: `frontend/schemas/`
- Backend URL resolver: `frontend/utils/backend-url.ts`

## UI and copy

- Reuse existing components and current DaisyUI-oriented patterns.
- Apply mobile-first layouts by default, then enhance for tablet/desktop breakpoints.
- Prefer responsive spacing/typography and avoid desktop-only assumptions in interaction flows.
- Use `lucide-react` icons.
- For user-facing medical text: avoid certainty claims; use plain language and clear next steps.
- Avoid the word "cluster" in user-facing text; use "group".

## Validation

- Minimum check for frontend code changes: `npx tsc --noEmit`
- Optional broader check: `npm run build`
