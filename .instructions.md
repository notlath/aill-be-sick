# AI'll Be Sick Copilot Instructions (Operational Overlay)

This file is the concise operating layer for day-to-day Copilot requests.
Use this together with `AGENTS.md` files, where `AGENTS.md` provides deeper reference details.

## Canonical architecture facts

- Monorepo with `backend/` (Flask) and `frontend/` (Next.js App Router).
- Backend local runtime: `http://localhost:10000` using `python run.py`.
- Frontend local runtime: `http://localhost:3000`.
- Backend URL handling in frontend is centralized in `frontend/utils/backend-url.ts` (includes localhost normalization behavior for server-side execution).

## Frontend implementation defaults

- Use TypeScript for all new frontend code.
- Use Server Components for data fetching when possible.
- Use Server Actions (`next-safe-action`) for mutations.
- Pair every mutation action with a Zod schema in `frontend/schemas/`.
- Use `actionClient.inputSchema(...).action(...)` from `frontend/actions/client.ts`.
- After mutation, apply cache revalidation with `revalidatePath` and/or `revalidateTag` as appropriate.
- Return structured success/error objects; avoid uncaught throws for expected failures.

## Backend implementation defaults

- Keep routes in Flask blueprints under `backend/app/api/`.
- Keep reusable logic in `backend/app/services/`.
- Read tunables/thresholds from `backend/app/config.py` and environment variables (avoid hardcoding threshold literals in endpoints).
- Keep endpoint failures explicit and safe for API consumers.

## UI and copy defaults

- Prefer existing reusable components and established DaisyUI patterns before introducing new UI patterns.
- Build UI changes mobile-first, then scale up to larger breakpoints.
- Keep tap targets and interactive controls usable on small screens.
- Use `lucide-react` for icons.
- For user-facing medical text: use plain language, calm tone, and avoid certainty claims.
- Avoid using the word "cluster" in user-facing text; use "group".

## Validation defaults

- Frontend quick validation: `npx tsc --noEmit` in `frontend/`.
- Backend tests: run targeted `pytest` files under `backend/tests/` relevant to the change.

## Skill selection hints

- Use `frontend/.github/skills/medical-diagnosis-actions/SKILL.md` for diagnosis and server-action implementation tasks.
- Use `frontend/.github/skills/clinical-copywriting/SKILL.md` for patient/clinician-facing copy changes.
- Use `backend/.github/skills/flask-diagnostic-api/SKILL.md` for Flask endpoint/service updates.
- Use `frontend/.github/skills/d3-viz/SKILL.md` for bespoke charting and map visualization work.
- Use `find-skills` when users ask to discover, recommend, or install additional skills.

- When frontend UI work includes component API/composition refactors, also apply `vercel-composition-patterns`.
