# AI Prompt Library (AI'll Be Sick)

Use these prompts directly with Copilot Chat for repeatable output quality.

## 1) Implement a frontend mutation (Server Action)

```text
Implement [feature] using the project Server Action pattern.
Requirements:
- Create/update Zod schema in frontend/schemas
- Use next-safe-action with actionClient.inputSchema(...).action(...)
- Keep error handling as structured success/error object
- Add revalidatePath/revalidateTag where needed
- Follow existing file naming conventions
Validation:
- Run npx tsc --noEmit in frontend and report result
Deliverables:
- Modified files list
- Why each file changed
- Validation output summary
```

## 2) Implement a backend Flask API enhancement

```text
Implement [feature] in Flask backend.
Requirements:
- Keep route logic in backend/app/api and reusable logic in backend/app/services
- Read thresholds/config from backend/app/config.py or env vars (no hardcoded threshold literals)
- Return explicit structured API responses for expected failure paths
- Keep current endpoint behavior backward-compatible unless explicitly requested
Validation:
- Run targeted backend tests for touched behavior
Deliverables:
- Modified files list
- Behavior changes and non-changes
- Test output summary
```

## 3) Fullstack bugfix workflow

```text
Fix [bug description] across frontend and backend.
Workflow:
1) Reproduce and identify root cause
2) Implement minimal fix at root cause
3) Avoid unrelated refactors
4) Validate with targeted checks
Deliverables:
- Root cause
- Code changes
- Validation evidence
- Remaining risks (if any)
```

## 4) Medical copy rewrite

```text
Rewrite this user-facing message for non-technical adults.
Constraints:
- Calm, supportive, plain language
- No absolute diagnosis claims
- Actionable next step guidance
- Avoid the word "cluster"; use "group"
Output:
- Revised copy
- 1-2 sentence rationale
```

## 5) PR-ready implementation request

```text
Implement [task] and prepare output in PR-ready format.
Include:
- What changed
- Files changed
- Testing done
- Risks and follow-ups
Also ensure changes respect AGENTS.md and .instructions.md guidance.
```

## 6) AI guidance alignment audit (instructions/prompts/skills)

```text
Audit whether our AI guidance files are still useful and aligned with the current codebase.

Scope files to audit:
- .instructions.md
- frontend/.instructions.md
- backend/.instructions.md
- .prompt.md
- COPILOT_CONFIG.md
- AGENTS.md
- frontend/AGENTS.md
- backend/AGENTS.md
- .github/pull_request_template.md
- frontend/.github/skills/**/SKILL.md
- backend/.github/skills/**/SKILL.md

Audit requirements:
1) Identify stale or contradictory statements (runtime, ports, framework, styling rules, data patterns).
2) Verify references to real files/functions still exist.
3) Check that recommended patterns match current code conventions in frontend/actions, frontend/schemas, backend/app/api, backend/app/services.
4) Flag missing guidance for new architecture patterns introduced since last update.
5) Propose minimal edits only; avoid rewriting whole docs unless necessary.

Output format:
- Alignment score (0-100)
- Top mismatches (highest risk first)
- File-by-file fix list (specific edits)
- Optional cleanup suggestions (low priority)

Validation:
- Run targeted grep/file checks to confirm each flagged mismatch.
```
