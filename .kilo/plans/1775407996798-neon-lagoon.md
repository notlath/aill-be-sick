## Overview
Implement age-based validation in the admin/clinician patient registration form at `/create-patient`. Calculate patient age from Date of Birth (DOB), allow normal registration for adults (â‰Ą18), and require guardian consent for minors (<18).

## Current State Analysis
- Patient registration form exists at `frontend/components/clinicians/users-page/create-patient-form.tsx`
- DOB field exists as required date input
- Server action `frontend/actions/create-patient.ts` handles registration
- Schema `frontend/schemas/CreatePatientSchema.ts` validates input
- No existing guardian consent logic or fields
- Database schema (Prisma) has no guardian fields in User model

## Proposed Implementation Plan

### 1. Frontend Form Updates
- Add age calculation utility function
- Modify form to dynamically show/hide guardian consent section based on calculated age
- Add guardian information fields (name, contact, relationship) for minors
- Add explicit consent confirmation checkbox for minors
- Prevent form submission if minor and consent not provided

### 2. Schema Updates
- Extend `CreatePatientSchema` to include optional guardian fields
- Add validation rules for guardian fields when patient is minor
- Add consent confirmation boolean field

### 3. Database Schema Changes
- Add guardian-related fields to User model in Prisma schema:
  - `guardianName` (String?)
  - `guardianEmail` (String?)
  - `guardianPhone` (String?)
  - `guardianRelationship` (String?)
  - `guardianConsentGiven` (Boolean? default false)
- Run Prisma migration to update database

### 4. Server Action Updates
- Add age calculation logic in action
- Validate guardian consent is provided and confirmed for minors
- Store guardian information in database for minors only
- Return appropriate error messages for validation failures
- Maintain existing permission checks

### 5. UI/UX Considerations
- Use clear visual indicators for minor/adult status
- Show guardian section with smooth transitions
- Include helpful text explaining requirements for minors
- Follow existing DaisyUI styling patterns
- Ensure mobile-responsive design

### 6. Testing & Validation
- Add unit tests for age calculation
- Test form validation for both adult and minor flows
- Verify database saves guardian info correctly
- Test permission checks remain intact
