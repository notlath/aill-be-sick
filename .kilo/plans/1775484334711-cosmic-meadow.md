# Plan: Implement Age-Based Guardian Consent for Patient Registration

## Overview
Implement conditional guardian information collection and consent requirement for patients under 18 years old during the create-patient flow. The system will calculate age from birth date and trigger guardian fields when age < 18.

## Current State Analysis
- Database schema already includes guardian fields (`guardianName`, `guardianEmail`, `guardianPhone`, `guardianRelation`)
- `CreatePatientSchema` has guardian fields commented out
- `createPatient` action has age calculation but guardian validation disabled
- Form component has guardian UI elements but conditional rendering disabled
- Age calculation utility (`calculateAge`) exists and works correctly

## Implementation Steps

### 1. Update CreatePatientSchema (frontend/schemas/CreatePatientSchema.ts)
- Uncomment and enable guardian fields: `guardianName`, `guardianEmail`, `guardianPhone`, `guardianRelation`
- Keep `guardianConsent` as optional boolean (will be required conditionally in action)
- Ensure all fields are properly typed

### 2. Modify createPatient Action (frontend/actions/create-patient.ts)
- Uncomment and enable guardian validation logic
- When `isMinor` (age < 18), require:
  - `guardianName` (non-empty)
  - `guardianEmail` (valid email)
  - `guardianRelation` (non-empty)
  - `guardianConsent` (true)
- Save guardian data to database when minor
- Return appropriate error messages for missing guardian information
- Update logging to include guardian validation status

### 3. Update CreatePatientForm Component (frontend/components/clinicians/users-page/create-patient-form.tsx)
- Enable conditional rendering of guardian section: `{age !== null && age < 18 && (`
- Uncomment guardian fields in form submission
- Ensure guardian state variables are properly reset on success
- Update form validation to require guardian fields when age < 18
- Display age information to clinician for transparency

### 4. Testing and Validation
- Test with birth dates resulting in age >= 18 (guardian section should not appear)
- Test with birth dates resulting in age < 18 (guardian section should appear and be required)
- Test edge cases: age exactly 18, invalid dates
- Verify form submission fails without guardian data when required
- Verify database saves guardian information correctly

### 5. Documentation Updates
- Update `docs/SYSTEM_PAGE_NAVIGATION_FLOWCHART.md` if registration flow changes affect navigation
- Update `docs/ACCOUNT_CREATION_FLOWCHART.md` to include guardian consent step

### 6. Code Quality Checks
- Run `npx tsc --noEmit` to ensure TypeScript compilation
- Verify schema validation works correctly
- Check that revalidation paths are correct
- Ensure no hardcoded threshold literals (age 18 is appropriate here)
